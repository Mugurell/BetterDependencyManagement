# Better dependency management in Android projects




Simple project that showcases better dependency management and can be used as a starter for any Kotlin based Android project.

Depends on the **buildScr** pattern for dependencies ([Caster.io lesson](https://caster.io/lessons/gradle-dependency-management-using-kotlin-and-buildsrc-for-buildgradle-autocomplete-in-android-studio)) that allow for *cleaner gradle configuration files, with autocomplete support for dependencies*.

Exemplifies the dependency implementation of 
* various common libraries like: Retrofit, Dagger2, RxJava2, etc
* various testing frameworks and libraries: Junit, Spek, Espresso, Mockito
* various static-code analysis tools: KtLint, Kluent    
and of course Jacoco
